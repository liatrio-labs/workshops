# Project Rules Generation

## Objective
Create comprehensive project rules and conventions based on requirements and architecture decisions.

## Process
1. **Analyze Requirements:** Review product requirements document and architecture decisions
2. **Identify Tech Stack:** Determine technologies, frameworks, and tools being used
3. **Define Standards:** Establish coding standards, naming conventions, and structure
4. **Create Rules Document:** Generate comprehensive rules in `.amazonq/rules/rules.md`

## Rules Categories to Address

### Directory Structure
- Source code organization
- Test file placement
- Documentation structure
- Configuration file locations

### Naming Conventions
- File and directory naming
- Variable and function naming
- Class and module naming
- API endpoint naming

### Code Standards
- Formatting and style guidelines
- Comment and documentation requirements
- Error handling patterns
- Testing requirements

### Git Workflow
- Branch naming conventions
- Commit message format
- PR/MR requirements
- Code review process

### Security Guidelines
- Authentication patterns
- Data validation rules
- Secret management
- Input sanitization

### Performance Standards
- Response time requirements
- Resource usage limits
- Caching strategies
- Optimization guidelines

## Output Location
Save all rules to: `.amazonq/rules/rules.md`

## Validation
- ✅ Rules are specific and actionable
- ✅ Standards align with tech stack best practices
- ✅ Guidelines support project requirements
- ✅ Rules are enforceable and measurable
