Write user stories and flows that describe what users need to accomplish. 

Objective:
- Stories should define the user journey through the application. 
- Take the application features into account and their relationships.
- Describe interactions with user flows.

Use this format for creating user stories:

```
As a [type of user],
I want [goal/desire]
so that [benefit/value]

Acceptance Criteria:
- Given [context]
- When [action]
- Then [outcome]
```

Organize by:
- User type or persona
- Feature area
- Priority (must-have, nice-to-have)
- Complexity (simple to complex)

Output: 
Create a prioritized list of user stories and flows that will guide development. Include enough detail to understand the requirement, but stay flexible on implementation.
